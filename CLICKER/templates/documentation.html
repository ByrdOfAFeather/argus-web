ARGUS Documentation

by Matthew Byrd (byrdof@ad.unc.edu), Tyson Hedrick

1Keyboard Shortcuts

ARGUS uses a variety of keyboard shortcuts to allow for speedy and accurate work.

When hovering over a video (notice a thin black border) with the page focused:

x - Zoom out

z - Zoom in

L - Lock to epipolar lines (This will let you only click points on the present epipolar lines)

SPACE - clicks a point with the location noted in the zoom window

ENTER - clicks a point with the location noted in the zoom window

Arrow Keys - Allows for precise movement of the zoom window (allows for precise clicking)

ALT + Arrow Keys - Moves the zoom window by a very small amount

SHIFT + Arrow Keys - Moves the zoom window by a large amount

3Saving/Loading

In the beta, saving and loading are done through json files. You can obtain a save file for your project with the save button . This can then be loaded from the main menu with the button labeled “continue working.”

4Track Mangement

The track management widget is located on the left of the workspace. Tracks are meant to track conceptually different things. For example, different body parts of a moving animal should be put on different tracks. A practically infinite number of tracks can be created, depending on the project, but after 5, you will need to use the buttons to navigate through the list of tracks. Each track has an associated color and visibility setting. The visibility settings will allow multiple tracks to be displayed at once (note that you can only edit one track at a time). The first icon is the edit icon; when this is pressed, you will be allowed to edit this track. The last icon is the delete icon; this deletes the track and all associated points from memory. There is no way to undo this, so be careful!

5Exporting Points

Exporting points can export a variety of point types depending on the settings. Since multiple .csv files could be created when exporting points, all files are placed into a .zip file before the download.

For 1-camera projects:

If you have the origin set (via the set-origin button), all points will be exported in their raw coordinates as well as their origin-based coordinates with proper units from the origin. If the origin is not set, only the raw coordinates are set. The raw coordinates have origin in the upper left corner with down, right being the positive direction. The exported points will have rows representing the frames of the videos and the columns being the x,y coordinates.

For 2-camera projects:

If you have DLT Coefficients loaded, 2D and 3D points will be exported. 3D points will only have non-null values for frames where every video has a point. The exported points will follow the same format mentioned above.

6Control Settings

The lock to DLT setting (L) will lock the zoom window to the DLT routines' lines. This will allow for more accurate clicking when using this setting. This feature will have no differences between enabled or disabled if DLT coefficients aren't loaded.

The forward/backward offset will determine how many frames the video moves forwards/backward after a point it clicked (if auto-advance is enabled) or after the f/b keys are pressed.

The auto-advance setting determines if a video will automatically move forward a frame after a point is clicked.

The sync setting determines if all videos will always be on the same frame; when this is disabled, you will have to manually move videos to the same frame if so desired.

7Video Settings

The video settings for individual videos can be accessed by pressing the gear icon at their associated title bar's top right. This will show a menu similar to the one seen during the initial project setup. Here you can change contrast, saturation, brightness, as well as offset, and colorspace. For more information, consult the tooltips provided.

8Popping out a video

Videos can be put into pop-out windows to make better use of screen real estate. The popped-out windows will be reloaded on the main page as soon as they are closed; it is typically very safe to use these without risking the page crashing. However, there are a few problems. There is a bug in the Vivaldi browser that will crash it when the webpage is closed without closing the popped-out windows first. For all other browsers, closing the main webpage will close all the pop-out windows.
